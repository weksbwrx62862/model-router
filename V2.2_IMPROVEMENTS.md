# Model Router v2.2 — 借鉴优化

## 新增能力

### 1. 成本监控

记录每次调用的 token 和成本，支持成本仪表盘查看。

**工具**：`cost_dashboard` — 查看模型调用成本统计

**日志文件**：`~/.hermes/model_router_costs.jsonl`

**记录内容**：
- 模型名称
- Provider
- 输入/输出 token 数
- 成本（美元）
- 延迟（毫秒）
- 任务类型和复杂度

### 2. 影子流量

1% 抽样对比质量，确保路由不会悄悄降低质量。

**日志文件**：`~/.hermes/model_router_shadow.jsonl`

**工作原理**：
- 每 100 次请求，抽 1 次同时调用更贵的模型
- 记录两个模型的响应
- 定期人工评估质量是否下降

### 3. 路由矩阵配置化

路由策略从代码中抽离，支持 JSON 热更新。

**配置文件**：`~/.hermes/routing_matrix.json`

**格式**：
```json
{
  "('classify', 'light')": "cheapest",
  "('classify', 'medium')": "cheapest",
  "('code', 'heavy')": "smartest",
  ...
}
```

**热更新**：每 60 秒自动刷新，无需重启。

### 4. 任务分类器

用最便宜的模型做前置分类，识别任务类型和复杂度。

**分类类型**：
- classify: 简单分类
- extract: 信息提取
- simple_qa: 简单问答
- long_doc: 长文档处理
- code: 代码生成/修改
- math: 数学计算/推理
- complex_reasoning: 复杂推理/分析
- agent: Agent 调用/工具使用

**复杂度**：light / medium / heavy

## 使用方式

### 查看成本仪表盘

```
/cost_dashboard
```

或通过工具调用：
```json
{"name": "cost_dashboard", "arguments": {}}
```

### 修改路由矩阵

编辑 `~/.hermes/routing_matrix.json`，60 秒内自动生效。

### 查看影子流量对比

```bash
cat ~/.hermes/model_router_shadow.jsonl
```
