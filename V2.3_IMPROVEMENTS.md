# Model Router v2.3 改进

## 新增功能

### 1. 长上下文自动绕路

**问题**：长文档/长对话场景下，常规路由可能选到 context_window 不够的模型，导致截断或失败。

**方案**：
- 每个模型增加 `context_window` 属性（从 config 或启发式推断）
- 估算输入 token 数量（中文约 1.2 token/字）
- 超过 200K tokens → 优选大上下文模型（`long_context_prefer`）
- 超过 800K tokens → 直选最大上下文模型（`long_context_bypass`）
- 跳过常规评分矩阵，直接返回

**阈值**：
```python
_LONG_CONTEXT_THRESHOLD = 200_000      # 优选大上下文
_VERY_LONG_CONTEXT_THRESHOLD = 800_000 # 直选最大上下文
```

### 2. 模型上下文窗口感知

每个模型现在携带 `context_window` 属性：

| 模型 | context_window |
|------|---------------|
| MiMo v2.5 Pro | 1,000,000 |
| MiMo v2.5 | 1,000,000 |
| MiMo v2 Omni | 256,000 |
| DeepSeek V4 Pro | 1,000,000 |
| DeepSeek V4 Flash | 1,000,000 |
| Llama 4 Maverick | 1,000,000 |
| Qwen Coder 480B | 262,144 |
| 其他 | 128,000 |

### 3. 长文档任务评分加成

`_score()` 函数新增长文档任务加成：
- `task_type == "long_doc"` 且 `context_window >= 1M` → +8 分
- `task_type == "long_doc"` 且 `context_window >= 256K` → +4 分

### 4. 路由日志增强

长上下文绕路时，日志中会显示：
```
[Model Router] 策略=auto | 长上下文绕路: 300,000 tokens | 选中: mimo-v2.5-pro (mimo) | 原因: 长上下文(300,000 tokens > 200,000)→优选大上下文模型
```
